    Overview

    This guide teach you how to combine javascript to a file and minify it. It could reduce file size and http request.

    The Building apply nodejs to build and conbine javascript. It load javascript according to AMD standard.

    Also combinte css, xml

    Quick Start

    Download nodejs and install nodejs. Open terminal and cd directory of amdquery project, cd build.

    Input "npm install" or "sudo npm install" in terminal before start building. NPM will download and install some node modules.

    Input "node build_js.js build_js_config.js" and watch the log.

    Look the directory: build/dist/defines. Look over "test.js" and "test.debug.js".

    Build Dependencies

    There are node modules to support Build. The "UglifyJS" be used to minify js and detect syntax error. The clean-css be used to minify css. The "xmldom" be used to parse HTML and compile HTML.

    If you want more infomation, please click the link and read "build_js.js" or "build_app.js".

    Anonymous Module Define

    If you define a module without name, build would add the name which is file name. So javascript can parse successive module by a combination js file.
    Best practices is defination the name of each module.
    Define a module using a relative path. Module path is relatived "amdquery.js".

    Overview

    This guide will teach you how to build application, How to configure build.
    You also need to know how to create an application.
    If you need more details or to change the build process, you need to have some knowledge about the nodejs and node-modules.
    If you want to build your own js file or css file, you can refer to uglifyOptions and cleanCssOptions.

    Quick Start

    Download nodejs and install nodejs. Open terminal and cd directory of amdquery project, cd build. Input "npm install" or "sudo npm install" in terminal before start building. NPM will download and install some node modules. Input "node build_app.js build_app_config.js" and watch the log. Look the directory: build/dist/apps.

      • name {String}

        The name of the application.

      • path {String}

        Application HTML path.

      • debug=true {Boolean}

        If that is true, that build will create a non-compressed "amdquery.debug.js", set the configuration of application to "true" in HTML

      • detectUIWidgetCSS=true {Boolean}

        If that is true, then build would detect widget-ui css from "appCombinationXMLPath".

        If that is false, then build would include all of "ui/css/*.css".

      • includeLibJSFromHead=true {Boolean}

        The lib js in head will be combinate and include in new HTML`s head if this is ture.

        The lib js must in head if not then build could not find the js.

      • aQueryConfig=false {Boolean|String}

        If it is a boolean, and it is true then it means you define "aQueryConfig" in inner script. Like this: 

        <script type="text/javascript">
var aQueryConfig = {
    amdquery: {
      define: "$",
      debug: false,
      development: true,
      loadingImage: ""
    },
    amd: {
      detectCR: false,
      debug: true,
      timeout: 5000,
      console: false,
    },
    amdVariables: {

    },
    ui: {
      initWidget: false,
      autoFetchCss: true,
      isTransform3d: true
    },
    module: {

    },
    app: {
      src: "",
      debug: false,
      development: true,
      autoFetchCss: true,
      viewContentID: ""
    }
  };
</script>

        If it is a string, it means you define "aQueryConfig" in a "*.js" and you add tag "script" to head before "amdquery.js"
        The path is relative to "app.html" path. 

        <script type="text/javascript" src="aQueryConfig.js"></script>
<script src="../amdquery/amdquery.js" type="text/javascript"></script>


        The most important is: If "aQueryConfig" is null, the tag "script" of amdquery.js must have an attribute which is "app". 

        <script src="../amdquery/amdquery.js" app="src:../document/app;debug:1;" type="text/javascript"></script>

        I support developer define config by attribute.

      • UIWidgetCSS=[] {Array}

        It is a array of "ui/css/some.css", for example: ["tabbar.css", "navmenu.css"]. The array will concat into build-list when building widget-ui css.

        Do not worry about duplicate css, build would filter same.

    Build Detail

    1.Detecting whether there is "aQueryConfig".

    Find aQueryConfig or not.

    2.Open HTML. Get "HTML" path from the configuration.

    Use "xmldom" to parse html, Get an array of css hrefs and script tag of application. Parse configuration of the tag.
    The config defined in script attributes is is the highest priority.

    3.Create application directory and copy file.

    Create application directory. In this example, path of directory is "dist/apps/document/". If the path of directory exists then remove it.
    Copy directory:
    (This example: AMDQueryJSRootPath is "/Users/apple/project/my/amdquery/")
    (This example: appProjectPath is "/Users/apple/project/my/amdquery/document/")
    (This example: distPath is "/Users/apple/project/my/amdquery/build/dist/apps/document/". "document" is application name)
    (This example: AMDQueryProjectPath is "/Users/apple/project/my/amdquery/")
    Copy AMDQueryProjectPath + "global" to distPath + ""
    Copy appProjectPath + "assets" to distPath + "app/assets"
    Copy appProjectPath + "styles" to distPath + "app/styles"
    Copy appProjectPath + "xml" to distPath + "app/xml"
    Copy appProjectPath + "lib" to distPath + "app/lib"
    Copy AMDQueryJSRootPath + "ui/css" to distPath + "amdquery/ui/css"
    Copy AMDQueryJSRootPath + "ui/images" to distPath + "amdquery/ui/images"

    4.Build library Javascript from HTML`head.

    Whatever you include a "src" of javascript or modify content in inner of javascript, build will get the content to combinate to two file and save it into directory "lib".
    One of files is "beforelib.js", another is "afterlib.js".
    If "Config.debug" is false then minify content. 

    <script type="text/javascript">
var applicationCache = null;
</script>
<script src="lib/aQueryConfig.js" type="text/javascript"></script>
//before amdquery.js
<script src="../amdquery/amdquery.js" type="text/javascript" app="src:../test/app;debug:1;"></script>
//after amdquery.js
<script type="text/javascript">
var bTest = "Test buildLibJSFromHead";
</script>
<script src="lib/aQueryConfig.js" type="text/javascript"></script>
//                        ||
//                        || After build
//                        \/
<script src="lib/beforelib.js " type="text/javascript"></script>
<script src="../amdquery/amdquery.js" type="text/javascript" app="src:../document/app;development:0;debug:true;viewContentID:aQueryViewContentKey;"></script>
<script src="lib/afterlib.js " type="text/javascript"></script>

    5.Build Javascript of Application.

    Get "app.js" path to the directory. Set "require" parameter with the path. ("@app" ==> "../document/app/app")
    "oye.node" could load all js after setting "require" parameter. Then build save the js files to the dist directory. You also can see then log in terminal.
    Save file: /Users/apple/project/my/amdquery/build/dist/apps/document/amdquery/amdquery.js
    Save file: /Users/apple/project/my/amdquery/build/dist/apps/document/amdquery/amdquery.debug.js

    6.Build XML of Application.

    Put all the XML merged into one XML. Root tag is "< div id='aQueryViewContentKey' >". Each original XML will be wrapped in "< aQueryViewContentKey='name' >" tag within
    The new XML will be saved as "combination.xml". "combination.xml" is easy for reading.
    Content of new XML will append into "app.html"

    Build "document" xml file

    Save app Combination xml: /Users/apple/project/my/amdquery/build/dist/apps/document/xml/combination.xml

    7.Build CSS of Application.

    After the first step has been to get all the "CSS" href, this step will be to find those CSS file through the href and use the "clean-css" to merge them into a CSS file.
    The new CSS will be saved as "{#appName}.css" moreover the final step will automatically add a "CSS" tag to "HTML" head, the properties href will be set "styles/{#appName}.css".

    Build "document" css file
    buildAppCss css path: /Users/apple/project/my/amdquery/document/styles/reset.css

    Save app Combination css: /Users/apple/project/my/amdquery/build/dist/apps/document/styles/document.css

    8.Build CSS of AMDQuery UI.

    Build combine all the CSS of AMDQuery UI and save it as "amdquery-widget.css".

    The priority of "UICssPath" which is defined in head is highest.
    Second priority is the css list form html defined by "amdquery-widget" or the css list from build directory("amdquery/ui/css/*.css").
    The lowest priority is css defined by "build_config".
    See the function "_buildUICss" of "build.js".

    Save UI amdquery-widget.css: /Users/apple/project/my/amdquery/build/dist/apps/document/amdquery/ui/css/amdquery-widget.css

    9.Modify HTML.

    Remove all CSS in head and add a new CSS into head, the new CSS href is "styles/document.css".
    Modify "amdquery.js" script attribute:
    change "src" as ".." + htmlInfo.appDirectoryName + "app"
    change "development" to "false"
    change "debug" as you define in configuration
    XML append into "app.html".

    APP

    Build set the configuration of the "viewContentID" property to "aQueryViewContentKey" and add combination "XML" into "app.html" body in order to synchronize the contents of loaded view.
    This property is used to tell the Application how to find the content view in a non-development model. After building the application development model is false.
    When the view is loading, it look for ID as "viewContentID" of "div" after look for property as "viewContentID" of sub-elements.

  3. cleanCss And uglify options

    uglifyOptions

    • strict_semicolons:Boolean[false]

      disable automatic semicolon insertion and support for trailing comma in arrays and objects.

    • mangle_options
      • mangle:Boolean[true]

        pass false to skip mangling names.

      • toplevel:Boolean[false]

        mangle toplevel names (by default we don’t touch them).

      • defines:Array[null]

        an object with properties named after symbols to replace.

      • except:Object[null]

        an array of names to exclude from compression.

      • no_functions:Boolean[false]

        no mangle functions

    • squeeze_options
      • make_seqs:Boolean[true]

        which will cause consecutive statements in a block to be merged using the “sequence” (comma) operator.

      • dead_code:Boolean[true]

        which will remove unreachable code.

      • no_warnings:Boolean[false]

        whether dist warnings.

      • keep_comps:Boolean[true]

        keep comps.

      • unsafe:Boolean[false]

        enable other additional optimizations that are known to be unsafe in some contrived situations, but could still be generally useful. For now only these:
        foo.toString() ==> foo+””
        new Array(x,…) ==> [x,…]
        new Array(x) ==> Array(x)

    • gen_options
      • indent_start:Number[0]

        (only applies when beautify is true) – initial indentation in spaces.

      • indent_level:Number[4]

        (only applies when beautify is true) – indentation level, in spaces (pass an even number).

      • quote_keys:Boolean[false]

        if you pass true it will quote all keys in literal objects.

      • space_colon:Boolean[false]

        (only applies when beautify is true) – wether to put a space before the colon in object literals.

      • beautify:Boolean[false]

        pass true if you want indented dist.

      • ascii_only:Boolean[false]

        pass true if you want to encode non-ASCII characters as \uXXXX.

      • inline_script:Boolean[false]

        pass true to escape occurrences of

    cleanCssOptions

    • keepSpecialComments:String["*"]

      "*" for keeping all (default), 1 for keeping first one only, 0 for removing all.

    • keepBreaks:Boolean[false]

      whether to keep line breaks (default is false).

    • processImport:Boolean[true]

      whether to process @import rules.

    • noRebase:Boolean[false]

      whether to skip URLs rebasing.

    • noAdvanced:Boolean[false]

      set to true to disable advanced optimizations - selector & property merging, reduction, etc.

    • selectorsMergeMode:String["*"]

      ie8 for IE8 compatibility mode, * for merging all (default).

  4. Jake

    • Overview
    • Quick Start
    • As a Contributor

    Overview

    Jake -- JavaScript build tool for Node.js
    Using Jake easy to build.

    Quick Start

    In terminal, cd build directory, "npm install" or "sudo npm install". Jake is automatically installed for global.
    Input "jake", you will see building options.
    jake buildapp[*.js(config file)]     build application
    jake buildapp     default is build_app_config.js
    jake buildjs[*.js(config file)]     build javascript
    jake buildjs     default is build_js_config.js
    jake jsdoc[default|amdquery(template name)]     build javascript api document
    jake jsdoc     default is amdquery
    jake ui_css     build css of widget-ui
    jake beautify[...file]     example 'jake beautify[a.html,b.css,c.xml,d.js]'

    As a Contributor

    As a contributor, if you modify javascript api or modify document, you should update the gh-pages.
    You can input "jake pages" to update gh-pages and you should know about this command will merge master and push gh-pages.
    You should read "jakefile.js" under directory build to know more detail.
